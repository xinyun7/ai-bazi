# 天机阁 · AI 八字算命系统说明文档

---

## 一、系统概述

**天机阁**是一个基于周易八字命理的 AI 运势分析 Web 应用。用户输入出生年月日、时辰和性别，系统自动计算四柱八字、五行分布，并通过 DeepSeek 大语言模型生成个性化的运势分析报告。

## 三、后端核心模块

### 3.1 server.js — 服务器与 API 路由

Express 5 应用，监听 `0.0.0.0:3000`（所有网络接口），主要职责：

- **静态资源托管**：`express.static('public')` 提供前端文件
- **速率限制**：`express-rate-limit` 对 `/api/` 路径限制每 IP 每分钟最多 10 次请求
- **DeepSeek 客户端初始化**：使用 OpenAI SDK，配置 `baseURL` 指向 DeepSeek
- **API 调用日志**：通过 `utils/logger.js` 记录每次 DeepSeek 请求参数、完整响应和耗时

提供以下 API 接口：

| 方法 | 路径             | 说明                                                      |
|------|-----------------|-----------------------------------------------------------|
| GET  | `/api/shichen`  | 返回十二时辰列表（名称 + 时间段描述）                        |
| GET  | `/api/categories` | 返回可分析的领域列表（综合/事业/财运/感情/健康）              |
| POST | `/api/bazi`     | 输入生辰，返回四柱八字、五行分布、日主分析（**纯计算，不调用 AI**）|
| POST | `/api/fortune`  | 输入生辰 + 领域，**SSE 流式返回**八字信息 + AI 运势分析文本    |

### 3.2 utils/bazi.js — 八字计算引擎

基于 `lunar-javascript` 库实现，核心功能：

1. **历法转换**：公历日期 → 农历日期（`Solar` → `Lunar`）
2. **四柱推算**：调用 `lunar.getEightChar()` 获取年柱、月柱、日柱、时柱的天干地支
3. **五行分布计算**：
   - 八个天干地支各对应一种五行（金木水火土）
   - 额外计入地支藏干（权重 0.5），使五行统计更精确
4. **日主强弱分析**：以日柱天干为日主，统计同类（比劫）和生我（印星）的五行力量占比，判定偏强/中和/偏弱
5. **大运方向计算**：阳年干男命/阴年干女命顺排，反之逆排
6. **流年信息**：获取当前日期的流年、流月、流日干支，用于近期运势分析

关键数据结构：

```javascript
// calculateBazi() 返回值
{
  fourPillars: {                    // 四柱
    year:  { gan, zhi, full },      //   年柱（天干、地支、完整干支）
    month: { gan, zhi, full },      //   月柱
    day:   { gan, zhi, full },      //   日柱
    time:  { gan, zhi, full },      //   时柱
  },
  dayMaster: { gan, wuxing, strength }, // 日主（天干、五行、强弱）
  wuxing: { 金, 木, 水, 火, 土 },       // 五行分布计数
  missingWuxing: ['水'],                // 缺失的五行
  lunarDate: { year, month, day },     // 农历日期
  shichen: { name, desc },             // 时辰信息
  liuNian: { ... },                    // 当前流年流月流日干支
  solarDate: { year, month, day },     // 原始公历输入
  gender: 'male' | 'female',          // 性别
  dayunDirection: '顺排' | '逆排',     // 大运方向
}
```

### 3.3 utils/prompt.js — Prompt 工程模块

负责构建发送给 DeepSeek 的消息数组（`messages`），由三部分组成：

1. **System Prompt**：设定 AI 角色为精通周易八字的国学大师，融合《滴天髓》《子平真诠》《穷通宝鉴》等经典著作，要求区分乾造/坤造分析
2. **八字信息格式化**：将 `calculateBazi()` 的结构化结果转为自然语言文本，包含命主信息、四柱、五行分布、日主、大运方向、流年信息
3. **领域分析模板**：5 套预定义的 Prompt 模板，每套规定了输出的章节结构：

| 领域       | 输出章节                                            |
|------------|-----------------------------------------------------|
| 综合运势   | 命局总评 → 近期运势 → 吉事预告 → 注意事项 → 开运指南 → 寄语 |
| 事业运势   | 事业命局 → 近期事业运 → 事业机遇 → 职场注意 → 事业建议    |
| 财运       | 财运命局 → 近期财运 → 进财机遇 → 破财风险 → 理财建议      |
| 感情运势   | 感情命局 → 近期感情运 → 桃花机缘 → 感情注意 → 感情建议    |
| 健康运势   | 健康命局 → 近期健康运 → 健康利好 → 健康警示 → 养生建议    |

最终通过 `buildMessages(bazi, category)` 组装为 OpenAI Chat 格式：

```javascript
[
  { role: 'system', content: '你是天机阁首席命理师...' },
  { role: 'user',   content: '八字信息 + 领域分析要求' },
]
```

### 3.4 utils/logger.js — API 调用日志模块

负责将每次 DeepSeek API 的调用详情写入文件，便于调试和审计。

**核心功能**：

- **按日期分文件**：日志文件存放在 `logs/` 目录，文件名格式为 `deepseek_YYYY-MM-DD.log`
- **自动创建目录**：首次运行时自动创建 `logs/` 目录
- **异步写入**：使用 `fs.appendFile` 异步追加，不阻塞请求处理
- **记录成功和失败**：正常调用记录完整请求参数和 AI 回复文本，异常调用记录错误信息

**日志条目结构**：

```json
{
  "timestamp": "2026-04-01T12:00:00.000Z",
  "duration_ms": 5230,
  "request": {
    "model": "deepseek-chat",
    "temperature": 0.85,
    "max_tokens": 3000,
    "messages": [{...}]
  },
  "response": "## 🔮 命局总评\n\n命主...",
  "error": null
}
```

**调用时机**（在 `server.js` 中）：

| 场景         | 记录内容                                    |
|-------------|--------------------------------------------|
| AI 分析成功  | 请求参数、完整 AI 回复文本、请求耗时（毫秒）    |
| AI 分析失败  | 请求参数、错误信息                            |

## 四、前后端数据交互

### 4.1 请求流程图

```
用户填写表单 → 点击"开始推演"
       │
       ▼
  POST /api/fortune
  Body: { year, month, day, shichenIndex, gender, category }
       │
       ▼
  服务端: 输入校验 → calculateBazi() → buildMessages() → DeepSeek API (stream)
       │
       ▼
  SSE 流式响应（text/event-stream）
       │
       ├─ data: { type: "bazi", data: {...} }       ← 八字计算结果（首包）
       ├─ data: { type: "content", data: "..." }    ← AI 文本片段（逐块推送）
       ├─ data: { type: "content", data: "..." }    ← ...（持续推送）
       └─ data: { type: "done" }                    ← 结束标记
```

### 4.2 SSE（Server-Sent Events）流式传输

系统采用 SSE 协议实现 AI 回复的实时流式展示：

**服务端**（`server.js`）：
- 设置响应头：`Content-Type: text/event-stream`、`Cache-Control: no-cache`、`Connection: keep-alive`
- 先发送一个 `type: bazi` 事件，包含完整的八字计算结果
- 逐块转发 DeepSeek 流式 API 返回的文本片段（`type: content`）
- 完成后发送 `type: done` 结束事件

**客户端**（`app.js`）：
- 使用 `fetch()` + `response.body.getReader()` 读取流
- 手动解析 SSE 格式（`data: {...}\n\n`），处理粘包和分包
- 收到 `bazi` 事件后渲染八字信息面板
- 收到 `content` 事件后累加文本，调用 `renderMarkdown()` 实时渲染（打字机效果）
- 收到 `done` 事件后移除打字光标，保存查询历史
- 支持通过 `AbortController` 中断请求

### 4.3 纯八字计算接口

`POST /api/bazi` 提供不依赖 AI 的八字计算能力（同步 JSON 响应），即使 DeepSeek API Key 未配置或余额不足，八字信息展示功能仍可正常使用。

### 4.4 数据格式参考

**请求体**（`POST /api/fortune`）：

```json
{
  "year": 1990,
  "month": 6,
  "day": 15,
  "shichenIndex": 6,
  "gender": "female",
  "category": "comprehensive"
}
```

**SSE 事件示例**：

```
data: {"type":"bazi","data":{"fourPillars":{...},"wuxing":{...},...}}

data: {"type":"content","data":"## 🔮 命局总评\n\n"}

data: {"type":"content","data":"命主庚午年生，日主为..."}

data: {"type":"done"}
```

## 五、历史查询结果存储

### 5.1 存储方案

系统使用浏览器 **localStorage** 存储查询历史，不依赖后端数据库。

| 项目         | 说明                                           |
|-------------|------------------------------------------------|
| **存储位置** | 浏览器 localStorage                              |
| **存储键名** | `tianjige-history`                              |
| **数据格式** | JSON 数组                                       |
| **容量上限** | 最多保留最近 **5 条**记录（FIFO，新记录插入首位）    |
| **存储时机** | AI 分析完成后（收到 `type: done` 事件时）自动保存    |
| **作用域**  | 同一浏览器 + 同一域名下共享，不跨浏览器或设备         |

### 5.2 存储的数据结构

每条历史记录包含用户的输入参数（不存储 AI 输出结果）：

```json
{
  "year": "1990",
  "month": "6",
  "day": "15",
  "shichenIndex": "6",
  "gender": "female",
  "category": "comprehensive",
  "timestamp": 1711843200000
}
```

### 5.3 历史记录功能

- **展示**：在输入表单下方显示历史记录列表，包含日期、时辰、性别、分析领域和查询时间
- **回填**：点击历史条目自动填充表单（年/月/日/时辰/性别/领域），用户无需重新输入
- **轮转**：超过 5 条自动淘汰最旧的记录

### 5.4 其他 localStorage 用途

| 键名                | 用途                     |
|--------------------|--------------------------|
| `tianjige-history` | 查询历史（上述）           |
| `tianjige-theme`   | 日/夜主题偏好（`light` 或 `dark`） |

## 六、前端 UI 设计

### 6.1 响应式布局

- **桌面端**（>768px）：内容居中，最大宽度 800px，卡片阴影效果
- **移动端**（≤768px）：全宽布局，缩小内边距，领域选择改为单列排列，四柱信息自适应

### 6.2 日/夜主题

通过 CSS 变量 + `data-theme="dark"` 属性切换：

- **日间模式**：米白背景 `#FFF8E7`、深褐文字 `#3E2723`、金色装饰 `#D4A574`
- **夜间模式**：深色背景 `#1a1a2e`、浅色文字、暗金装饰

用户偏好保存在 localStorage，刷新页面后保持。

### 6.3 关键交互

- **打字机效果**：AI 回复逐字渲染 + 闪烁光标动画
- **Markdown 渲染**：自带轻量 Markdown → HTML 转换（标题/粗体/列表/分隔线）
- **自动滚动**：AI 输出过程中自动滚动到内容底部
- **Toast 提示**：操作反馈（复制成功、错误提示等）
- **加载动画**：太极旋转 CSS 动画

## 七、安全设计

| 措施                     | 说明                                                  |
|-------------------------|-------------------------------------------------------|
| **API Key 保护**         | 存放在服务端 `.env` 文件，不暴露到前端                    |

## 九、启动与部署

### 本地运行

```bash
# 安装依赖
npm install

# 配置环境变量
# 编辑 .env 文件，填入 DEEPSEEK_API_KEY

# 启动服务
node server.js
# 或双击 start.bat

# 访问
# http://127.0.0.1:3000
```

### 停止服务

```bash
# 终端按 Ctrl+C
# 或双击 stop.bat
```

### Windows 一键脚本说明

- `start.bat`：启动前自动完成环境检查（Node.js、依赖包、.env 配置），并清理已占用端口，然后提供三种操作：
  - 选项 1：仅本地启动（`http://127.0.0.1:3000`），启动成功后自动打开浏览器
  - 选项 2：本地服务 + 公网隧道（自动运行 `tunnel.js`，生成公网访问地址）
  - 选项 3：停止所有服务（调用 `stop.bat`）
- `stop.bat`：同时停止本地 Node 服务和隧道进程

### 公网发布（最简单低成本）

系统已内置 `localtunnel` 方案，无需购买服务器即可临时发布到互联网：

1. 启动 `start.bat`
2. 输入 `2` 选择“本地 + 公网隧道”
3. 在隧道窗口复制 `https://xxxx.loca.lt` 公网地址分享给他人

注意事项：

- 关闭隧道窗口后公网访问会断开
- 首次访问 `loca.lt` 地址可能出现确认页面，点击继续即可
- 免费隧道地址可能变化，建议用于演示或小规模临时访问

### 环境变量

| 变量名               | 必填 | 说明                                    |
|---------------------|------|-----------------------------------------|
| `DEEPSEEK_API_KEY`  | 是   | DeepSeek 平台 API Key                    |
